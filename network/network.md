
# Network (네트워크)

  

## 📌 HTTP 기초


### NET-001

HTTP에 대해 설명해 주세요.

- Hyper Text Transfer Protocol 의 약자
- 브라우저에서 데이터를 주고받는 프로토콜

### NET-002

HTTP의 특성인 Stateless에 대해 설명해 주세요.

- Http 프로토콜은 무상태성을 특징으로 가지며 요청의 상태를 유지하지 않는다.
- 매 요청은 새로운 상태를 갖는다.

### NET-003

Stateless와 Connectionless에 대해 설명해 주세요.

- Statelesss는 해당 요청의 이전 상태와 현재 상태를 관리하지 않는것
	- 매 요청을 독립적으로 처리한다.
	- scale이 쉬움
- Connectionless는 네트워크 통신 커넥션을 유지하지 않는것
	- HTTP 1.0

### NET-004

왜 HTTP는 Stateless 구조를 채택하고 있을까요?

- Stateful한 프로토콜을 사용하면 상태유지해야함. 스케일이 어려워짐
- 장애 내성이 강해짐. 각 요청은 다른 컨텍스트기 때문에 서버하나 죽어도 다른 서버에서 처리가능해짐.
- 동기화 문제가 없음
### NET-005

Connectionless의 논리대로면 성능이 되게 좋지 않을 것으로 보이는데, 해결 방법이 있을까요?

- Http Keep Alive라는 헤더를 추가하면됨.
	- Http 1.0에서 사용
	- Keep Alive 시간이 지나면 서버가 닫아버림
	- 근데 프록시 서버때문에 문제가 생김
- Http 1.1부터는 기본이 Persistent connection

### NET-006

TCP의 keep-alive와 HTTP의 keep-alive의 차이는 무엇인가요?

- HTTP keep alive는 커넥션을 닫지 않고 재사용하는거
- TCP keep alive 는 연결이 살아 있는지 확인하는거
	- 폴링 하트비트로 확인

## 📌 HTTP 버전 및 성능

  

### NET-007

HTTP/1.1과 HTTP / 2 의 차이점은 무엇인가요?

- 1.1은 PlainText기반
	- FCFS 방식이라서 HOL 발생함
- 2.0은 기본적으로 바이너리임
	- 패킷을 짤라서 보냄 -> 프레임 단위라고 부름
	- 파이프라인, 멀티플렉싱 가능함
	- 서버 푸시 가능
	- 헤더 압축
### NET-008

HOL Blocking 에 대해 설명해 주세요.

  - 직렬성이 보장되는 작업에서 앞에 작업이 오래걸리면 전체 작업이 밀리는 현상

### NET-009

HTTP/3.0의 주요 특징에 대해 설명해 주세요.
  - QUIC 프로토콜 채택
	  - UDP 기반으로 작동
	  - 레이턴시를 최대한 줄이는 방식 RTT = 0
	  - Stream 단위의 독립 처리 가능
	  - 패킷을 잃어버려도 커넥션에 영향 없음.
	  - HTTP2의 TCP HOL 해결
- TCP HOL은 TCP 특성상 하나의 패킷이 누실되면 다시 보내야하는 현상임
	- 그동안 전체 TCP 커넥션이 중단됨.
	- 패킷 손실률이 올라가면 성능이 저하됨.

- QUIC의 독립 스트림
	- 스트림이 독립적이라서 하나의 링크가 죽어도 다른 스트림에 영향 없음
- QUIC는 TLS를 직접 사용함
	- HTTP 처럼 쓰거나 안쓰거나가 아님
	- TLS를 캡슐화해서 항상 암호화됨.
	- 핸드셰이크도 없어서 커넥션 맺는게 빠름

### NET-010

왜 HTTP는 TCP를 사용하나요?

- 안정적임.
- 메세지 기반의 통신이다보니 데이터 무결성이 중요함.
  

### NET-011

그렇다면, 왜 HTTP/3 에서는 UDP를 사용하나요? 위에서 언급한 UDP의 문제가 해결되었나요?

  - UDP는 기본 구현 자체가 포트빼면 아무것도 없는 베어메탈느낌이라 커스텀하게 만들기가 용이했음.
  - TCP의 확장기능을 만들기가 편함.
  - QUIC프로토콜이 신뢰성을 보장하고
  - 순서 보장 같은건 독립 스트림이 보장함
  
  [ ] QUIC가 항상 빠른가? 
  

### NET-012

그런데, 브라우저는 어떤 서버가 TCP를 쓰는지 UDP를 쓰는지 어떻게 알 수 있나요?

  - ALPN 사용
	  - TLS 핸드셰이크할때 정함
- 헤더
	- 헤더에서 HTTP/3지원하는지 알려줌

  
## 📌 서브넷 및 NAT

  

### NET-097

서브넷 마스크와, 게이트웨이에 대해 설명해 주세요.

- IP 주소 + 서브넷 마스크로 네트워크 주소를 얻는다.
- 네트워크랑 호스트 영역을 분리할 수 있음.
	- 255.255.255.0
	- 192.168.1.100
	- 이면 호스트는 AND연산때린거로 100나옴
- 같은 네트워크에 속한 기기들을 구분함.

게이트웨이
- 로컬 네트워크랑 외부 네트워크를 중계함.
- 패킷을 외부로 전송할때,  게이트웨이를 통해서 패킷을 전달


### NET-098

NAT에 대해 설명해 주세요.

사설 IP를 공인 IP로 바꾸는 기술
- Network Address Translation
- 사설 대역을 인터넷에서 사용 불가능함.
- 공인 IP는 하나를 쓰고 여러 기기가 내부 네트워크에서 사용됨

매핑 방식에 따라 달라짐.
- 정적 매핑 : 정적으로 할당
- 동적 매핑 : 공인 IP풀에서 할당
- 포트번호로 매핑

### NET-099

서브넷 마스크의 표현 방식에 대해 설명해 주세요.

- 십진수로 표기하거나 CIDR로 표기
- x.x.x.x/24
- 24개 까지 1
### NET-100

그렇다면, 255.0.255.0 같은 꼴의 서브넷 마스크도 가능한가요?

- 안되는건 아닌데 표준에 어긋남
- 1이 이어져야함.
- prefix가 모두 1이어야한다.
